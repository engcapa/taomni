0. 前置准备

工具链

  • Node 18+、pnpm、Rust 1.77.2+、Tauri 2 系统依赖（Win 上 WebView2，Linux 上 webkit2gtk）
  • 可选：Vulkan ICD（NV/AMD/Intel 驱动自带就有）；macOS 自带 Metal
  • API key（任选，测试用临时额度即可）：DeepSeek、GLM-4-Flash（免费）、SiliconFlow、Groq、Tavily / Serper（可选）

构建分两类，按测试目标选

场景命令
开箱即用冒烟（cloud-only）pnpm tauri dev
启用真 ASR + 真 FIM（重型）pnpm tauri dev --features voice-capture,asr-sherpa-onnx,local-llm-fim,vulkan-detect（pnpm 透传到 cargo 需要在 src-tauri/Cargo.toml 或 tauri-build 中显式传 -F，详见下文 §3 注释）
纯 Rust 单元/集成测试cargo test --manifest-path src-tauri/Cargo.toml
性能基线集成测试cargo test --manifest-path src-tauri/Cargo.toml --release perf_baseline -- --ignored
前端单测pnpm test（vitest）
UI 自动化qa-ui-auto skill 跑 qa-ui-auto-tests/cases/TC-AI-00*.testcase.yaml

Cargo features 不是默认开启的（default = []）。不带 feature 时 ASR / FIM 走"占位实现"——能发指令、能测路由，但出不了真转写/补全。先 cloud-only 跑通再加真模型，能避免一次性把"配置错"和"特性没启用"两类故障耦合在一起。

清理基线（每次大版本回归建议）

rm -rf ~/.config/newmob/ai.json ~/.cache/newmob/  # Linux/macOS
# Windows: del %APPDATA%\newmob\ai.json & rmdir /s /q %LOCALAPPDATA%\newmob

1. 第一层：cloud-only 开箱即用（80 MB，最快路径）

目标：纯云 LLM + 不开语音 / 不开 FIM，验证"装好就能聊"。

配置

  1. pnpm tauri dev 启动
  2. 设置 → AI → LLM Providers，选 glm（完全免费），粘贴 GLM API key，点"测试连接" → 期望 200 OK 绿点
  3. 顶部 LLM 段从 deepseek 切到 glm（或填 deepseek key 也行）
  4. ASR、FIM、Web Search 全部不开

冒烟点

  • Ctrl+L 打开 Drawer → 输入 say hello in 5 words → 流式 token 在 1 s 内开始出现，最终回答完整
  • 状态栏：ASR ○（未启用）、LLM: glm ✓、AI 写: 关、Search: 关、CC: ○
  • 右键终端 → AI: 解释最近的终端输出 → Drawer 自动新建 thread + 输出
  • 先关 Drawer，再开 → 上次 thread 仍在
  • 全本地模式开关打开 → glm 调用立刻被网络策略拒绝（Drawer 中报错"remote provider blocked by full-local policy"）；关回 → 恢复

自动化：cargo test --test cloud_only_smoke --manifest-path src-tauri/Cargo.toml

2. 第二层：语音（ASR 80 MB + 真 cpal 录音）

模型准备

  • 设置 → AI → 模型库 → Streaming Zipformer (zh-en, small) → 点"下载"
  • 下载到 <cache>/newmob/models/asr_sherpa_zipformer_zh_en_small/sherpa-onnx-streaming-zipformer-bilingual-zh-en-small.tar.bz2
  • 下载完成后在面板上 → "校验" 期望 SHA-256 通过

注意：models.manifest.json 当前 SHA-256 全是 0000…，校验逻辑遇到 placeholder 会跳过强校验。正式回归前先把 manifest 中的真实 SHA-256 填上——否则镜像被劫持也发现不了。手工算一次：sha256sum <下载路径>。

构建

必须带 feature 才有真转写：

cd src-tauri && cargo build -F voice-capture,asr-sherpa-onnx

不带 voice-capture 时 voice_capture_supported 返回 false，TitleBar PTT 按钮置灰。

配置

  • 设置 → AI → ASR Panel → active = sherpa-zipformer-zh-en
  • LLM 任务路由 voice_intent 仍走 glm/deepseek（云 + 文本，安全）

冒烟点

  • TitleBar 🎤 按下不放 → 红点亮 → 说"列出所有会话" → 松开 → ~1 s 内 Drawer/ActionCard 出现转写文本 + 路由后的工具调用建议
  • 中文说"切换到 prod 标签" → 转写中文 + 意图识别为 switch_tab
  • 短音频 < 0.5 s → 期望转写为空，不应触发任何 LLM 请求（节流测试）
  • 全本地模式打开后再 PTT → ASR 仍工作，但 LLM 调云端被拒绝；切到 active=local + sidecar 已起来后能恢复

自动化：voice_intent_latency.rs（用 fixture wav；可在 src-tauri/tests/fixtures/voice/ 放真录音）

3. 第三层：本地 LLM sidecar + Tab 补全 FIM（~2 GB）

模型准备（按需选一档）

用途模型 ID大小
Sidecar 二进制sidecar_llama_server_<triple>~25 MB
FIMllm_qwen3_0_6b_q4_k_m400 MB
默认 chat（可选）llm_qwen3_1_7b_q4_k_m1.4 GB

设置 → AI → 模型库 → 逐个下载；models_verify 全过；二进制落到 <cache>/newmob/binaries/sidecar_llama_server_*/llama-server[.exe]

配置

  • AsrPanel 旁边 → "本地 sidecar" → 点"启动" → sidecar_status 返回 running:true, base_url:http://127.0.0.1:8080/v1, gpu_backend:Vulkan|Metal|Cpu, binary_installed:true
  • LLM Providers Panel → local runtime=llama-server、model=qwen3-1.7b-q4_k_m，测试连接走 127.0.0.1
  • 任务路由：tab_completion = local、command_rewrite = deepseek（保留云端，FIM 留给本地）

构建

FIM 真路径必须 local-llm-fim feature；首次构建 llama-cpp-2 在 Windows 需要 clang/LLVM，大约 3–6 分钟。

冒烟点

  • 终端中输入 git che 暂停 200 ms → 灰色 ghost-text 显示 ckout main 之类候选；按 → 接受
  • 输入 mysql -ppassword=secret 后回车，再输入 mysql -p，FIM 不应把 password=secret 补回来（redact 验证）
  • Ctrl+K 弹改写小窗，输入 make it use --namespace=prod → 接收 diff
  • PowerShell 中开 ghost-text → 应不显示（PSReadLine 互斥例外保留）
  • Vim / less / top（alt-screen）下 ?? 起首 → 不拦截（直接进 vim 缓冲）
  • 关掉 sidecar → 走 fallback：tab_completion 任务期望 8 s 超时后抛错（不参与 fallback，router.rs 中是硬约束）

关键延迟基线（用 cargo test --release fim_latency）

指标目标 P95
历史 ghost-text< 10 ms
PATH 数据源< 50 ms
FIM in-process< 300 ms
Ctrl+K 改写（云端）< 2 s

4. 第四层：语音→Shell + Agent + 命令预览卡

前提：第二层语音可用 + LLM 配好

冒烟点

  • 设置 → AI → 实验性 → 启用"语音生成命令"（首次有一次性说明弹窗）
  • PTT："列出当前目录大于 100M 的文件" → 命令预览卡显示 find . -size +100M、风险 low、1.5 s 后自动注入终端
  • "删除所有日志" → 风险 medium → 必须 Enter
  • "把根目录全删了" → 黑名单命中（rm -rf / 正则）→ 卡片可见但 Enter 不执行，审计落 outcome=blocked_blacklist
  • 在某 SSH 会话上勾选"禁用 AI 写动作" → 同样指令 Enter 仅复制到剪贴板，不写 PTY
  • 报错命令（git push --force-with-lease 之类）后 → 状态栏"AI 解释" → ActionCard 弹"读取最近 50 行" → 允许 → Drawer 流式解释 + 修复命令按钮

Agent 多步（opt-in）

  • Drawer 中："找出 prod-web 上 /var/log/nginx/error.log 最近 100 行的 5xx 统计"
  • 期望调用链：list_sessions(query=prod-web) → switch_tab → run_in_terminal(dry_run=true) → 卡批准 → 执行 → explain_error
  • 单 thread 工具调用 ≤ 5 步，超过自动 stop

MCP server（opt-in）

  • 设置开 mcp_server_start → 拿到 127.0.0.1:<rand> + token → 用 curl -H "Authorization: Bearer <token>" http://127.0.0.1:<port>/tools/list 应返回工具列表；没 token → 401

自动化：单测 cargo test shell_safety::blacklist（100% 黑名单覆盖）+ safety::write_tools_default_dry_run

5. 第五层：Web Search 双轨

配置

  • 路径 A（无原生）：用 deepseek 作 active LLM，开 SearXNG（默认无 key 能用）→ 在 Drawer 问 "kubectl 1.32 release notes" → 弹"Web Search 确认卡"（关键词 + 提供方）→ 允许一次 → SearchProgressChip → 5 条结果 → AI 综合回答
  • 路径 B（原生）：active 切到 OpenAI/Claude/Gemini，开 prefer_native_web_search=true → 同样问题 → 不弹卡，Drawer 中渲染"🔍 搜索 N 条 (via openai)"紧凑 pill；客户端工具改名暴露为 deep_search
  • BYOK：填 Tavily key → 验证它落到 OS keyring（keyring_get 命令查一下）+ 重启不丢
  • SSRF 守门：web_fetch http://localhost:8080 → 拒；web_fetch ftp://... → 拒；只接受 https + 公网 IP
  • 三档隐私：per_call / per_thread / always / disabled 切换后行为符合预期；disabled 时 Agent 收到"工具不可用"

自动化：qa-ui-auto-tests/cases/TC-AI-003-web-search-client.testcase.yaml 与 TC-AI-004-web-search-native.testcase.yaml

6. 第六层：Claude Code 集成（可选）

模型准备

  • winget install Anthropic.Claude 或 npm i -g @anthropic-ai/claude-code
  • claude login 完成认证
  • claude --version 在 PATH 中可见

冒烟点

  • 设置 → 高级后端 → 启用 → 探针通过 → Drawer 顶部多出 Claude Code (本机) 选项
  • thread 切到 CC → "分析 src/components/terminal/TerminalPanel.tsx 的事件流" → CC 调 Read/Grep → 每次工具调用弹 NewMob ActionCard（permissions_mcp 收口验证）
  • CC 试图执行 rm -rf / → §5.3 黑名单在 permission_prompt 层拦下
  • CC 试图读 ~/.ssh/id_rsa → --add-dir 白名单 + deny 列表双重拒绝
  • 关闭 NewMob 再开同 thread → --resume 续接（看 cc_session_id 字段是否复用）
  • 全本地模式打开 → CC 选项整段隐藏

9. 常见踩坑

  • Tauri sidecar bundle：tauri.conf.json 里 bundle.externalBin 现在已不打包 sidecar，运行期从 <cache>/newmob/binaries/ 读。开发模式（pnpm tauri dev）首次启动如果没下过二进制 → sidecar_status.binary_installed:false → 必须先去模型库下载，不会自动从源码树拷
  • manifest SHA-256 placeholder：当前全是 0000…，正式回归前必须填真值（否则下载层会跳过完整性校验）
  • Vulkan 检测 panic：ash crate 在某些驱动上加载会 panic，代码已用 catch_unwind 包住；若状态栏一直显示"GPU 不可用"且控制台无错，重装 GPU 驱动到 ≥ NV 535 / AMD 24.0
  • Windows llama-cpp-2 编译慢/失败：必须装 LLVM/clang 并在环境变量里能找到 clang.exe；CI 第一次会拖到 5 分钟以上
  • GLM key 在 ai.json 里能看到明文：保存时前端会自动转 vault:<id> 引用，看 ~/.config/newmob/ai.json 里如果还是明文说明 vault 没解锁——先在设置里设一次 master password
  • chat-stream 事件丢包：Drawer 流式断断续续 → 多半是 chat_stream 命令失败回退到非流式 chat_send，看控制台 console 应有 warning

跑完 Phase 0–4 + 边界回归后，应用就具备发版前的最低质量信心；CC 与 Web Search BYOK 视下游用户群是否需要。
